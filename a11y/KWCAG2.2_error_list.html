한국형 웹 콘텐츠 접근성지침 2.2 검사항목과 오류 유형

  1. 인식의 용이성 (Perceivable) : 모든 콘텐츠는 사용자가 인식할 수 있어야 한다.

    1. (적절한 대체 텍스트 제공)

    텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다.

    1. 1-1 <img>, <input type=image> 등 이미지 요소에 대해 alt 속성을 제공하지 않거나 alt 속성은 선언되었으나, 그 내용이 img의 내용을 대체할 수 없는 경우와 오타로 표기되어 의미가 변경된 경우
    2. 1-2 불릿(Bullet)등 의미 없는 이미지에 alt 속성을 제공하지 않거나 속성 값을 빈값(alt="")으로 제공 하지 않는 경우 또는 불필요한 대체 텍스트를 제공한 경우
    3. 1-3 조직의 관계나 프로세스 등 복잡한 이미지의 내용을 대체 텍스트로 설명하였으나, 순서가 맞지 않 거나 설명이 부족하여 원래 이미지의 내용을 대신할 수 없다고 판단되는 경우
    4. 1-4 img 요소의 대체 텍스트를 title만으로 제공하는 경우
    5. 1-5 QR코드 등과 같이 시각적인 정보만으로 확인이 가능한 콘텐츠(기능)의 이동 주소 정보 등을 대체 텍스트 또는 설명, 링크 등으로 제공하지 않은 경우
    6. 1-6 배경 이미지가 의미 있는 정보를 제공하고 있으나, 이를 텍스트 형태의 정보로 제공하지 않은 경우
    7. 1-7 사용자가 직접 대체 텍스트가 필요한 콘텐츠를 등록할 때(예: 게시판에 콘텐츠 등록), 해당 콘텐츠 에 대한 대체 콘텐츠를 입력하는 기능을 제공하지 않은 경우
    8. 1-8 longdesc 속성을 이용했으나, 파일이 연결되지 않은 경우 또는 longdesc로 연결된 파일의 내 용이 longdesc 속성을 부여한 img가 가진 내용의 의미나 기능을 파악하기 어려운 경우
    9. 1-9 HTML5에서 img 요소에 longdesc 속성과 값을 사용한 경우
    10. 1-10 <area> 요소나 longdesc 속성을 사용하였으나 area나 longdesc의 서술 대상인 <img> 요소 자체에 alt 속성을 적절하게 제공하지 않은 경우
    11. 1-11 유의미한 소리 정보 등에 대한 대체 텍스트를 미제공한 경우

    2. (자막 제공)

    멀티미디어 콘텐츠에는 자막, 대본 또는 수어를 제공해야 한다.

    1. 2-1 멀티미디어 콘텐츠에 대체 수단(자막, 대본, 수어 중 하나 이상)을 전혀 제공하지 않은 경우
    2. 2-2 멀티미디어 콘텐츠에 대한 대체 수단의 내용을 불충분 또는 적절하지 않게 제공한 경우
    3. 2-3 사용자가 등록한 멀티미디어 콘텐츠에 대해 관리자가 대체 수단 입력 기능을 제공하지 않은 경우

    3. (표의 구성)

    표는 이해하기 쉽게 구성해야 한다.

    1. 3-2 표의 제목과 요약정보가 불충분하거나 적절하게 제공되지 않은 경우
    2. 3-3 제목 셀과 내용 셀을 구분하지 않거나 잘못 구분한 경우
    3. 3-4 다단, 병합 등 복잡한 표의 제공 시 headers, id 속성을 통해 제목 셀과 내용 셀을 연결하지 않은 경우
    4. 3-5 표의 정보를 이해하기 어려운 중첩 표(표 안의 표)를 넣어 제공한 경우
    5. 3-6 배치용 테이블에는 <th>, <caption> 요소, summary 속성을 사용하는 경우

    4. (콘텐츠의 선형구조)

    콘텐츠는 논리적인 순서로 제공해야 한다.

    1. 4-1 콘텐츠의 순서를 논리적으로 제공하지 않아 전후 맥락을 인식할 수 없는 경우
    2. 4-2 콘텐츠의 구조를 논리적으로 제공하지 않아 계층 관계를 인식할 수 없는 경우

    5. (명확한 지시사항 제공)

    지시사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다.

    1. 5-1 색, 크기, 모양, 방향 등 지시사항을 시각정보로만 제공한 경우
    2. 5-2 소리 등 지시사항을 청각정보로만 제공한 경우

    6. (색에 무관한 콘텐츠 인식)

    콘텐츠는 색에 관계없이 인식될 수 있어야 한다.

    1. 6-1 색상만으로 내용을 구분하거나 인식하도록 제공된 콘텐츠(그래프, 차트, 지도, 필수 입력항목 등)를 제공한 경우
    2. 6-2 활성/비활성 상태를 색으로만 구분하는 경우

    7. (자동 재생 금지)

    자동으로 소리가 재생되지 않아야 한다.

    1. 7-1 웹페이지에서 자동으로 재생되는 3초 이상의 배경음 콘텐츠를 제공하는 경우
    2. 7-2 마우스 오버 또는 키보드 초점을 받아 자동으로 배경음이 3초 이상 실행되는 경우

    8. (텍스트 콘텐츠의 명도 대비)

    텍스트 콘텐츠와 배경 간의 명도 대비는 4.5 대 1 이상이어야 한다.

    1. 8-1 보통 크기(18pt 미만, 또는 굵은 14pt 미만)의 일반 텍스트가 4.5:1을 만족하지 않은 경우
    2. 8-2 텍스트의 크기가 18pt 이상, 또는 굵은 14pt 이상의 텍스트가 3:1을 만족하지 않은 경우
    3. 8-3 이미지 텍스트가 폰트 종류와 상관없이 14pt 크기에 해당하는 18.66px 미만일 때 4.5:1 이상 을 만족하지 않은 경우, 또는 18.66px 이상일 때 3:1 이상을 만족하지 않은 경우(굵은 14pt 기 준은 적용하지 않음)

  2. 운용의 용이성(Operable) : 사용자 인터페이스 구성요소는 조작 가능하고 내비게이션 할 수 있어야 한다.

    9. (콘텐츠 간의 구분)

    이웃한 콘텐츠는 구분될 수 있어야 한다.

    1. 9-1 이웃한 콘텐츠를 다음 1가지 방법 이상으로 구별하지 않은 경우
      1. 1) 테두리를 이용하여 구분
      2. 2) 콘텐츠 사이에 시각적인 구분선을 삽입하여 구분
      3. 3) 서로 다른 무늬를 사용하여 구분
      4. 4) 콘텐츠 배경색 간의 명도와 채도를 달리하여 구분
      5. 5) 줄 간격 및 글자 간격을 조절하여 구분
      6. 6) 기타 시각적으로 콘텐츠를 구분할 수 있는 방법 등을 사용하여 구분

    10. (키보드 사용 보장)

    모든 기능은 키보드만으로도 사용할 수 있어야 한다.

    1. 10-1 키보드로 접근 및 운용이 불가능한 경우

    11. (초점 이동과 표시)

    키보드에 의한 초점은 논리적으로 이동해야 하며, 시각적으로 구별할 수 있어야 한다.

    1. 11-1 초점의 이동순서가 논리적이지 않으며 일관성이 없는 경우
    2. 11-2 초점 또는 키보드의 위치를 나타내는 요소가 시각적으로 표시되지 않은 경우
    3. 11-3 텍스트 등 운용할 수 없는 요소에 초점이 되는 경우
    4. 11-4 좌·우 쓸기(이전항목, 다음 항목)동작을 할 경우, 초점의 이동순서가 논리적이지 않으며 일관성이 없는 경우
    5. 11-5 화면에 보이지 않거나 의미 없는 개체에 초점이 이동되는 경우
    6. 11-6 키보드 조작이 불가능한 상태가 되어 다음 사용자 입력 또는 컨트롤 등으로 초점을 이동할 수 없거나 이전 페이지로 초점을 이동할 수 없는 경우

    12. (조작 가능)

    사용자 입력 및 콘트롤은 조작 가능하도록 제공되어야 한다.

    1. 12-1 컨트롤의 크기가 대각선으로 6.0mm 이상 제공하지 않은 경우
    2. 12-2 링크, 사용자 입력, 기타 컨트롤의 테두리 안쪽으로 1픽셀 이상의 여백을 제공하지 않은 경우

    13. (문자 단축키)

    문자 단축키는 오동작으로 인한 오류를 방지하여야 한다.

    1. 13-1 단일 문자 단축키(대/소문자, 구두점, 기호 등 글자키, 숫자키, 특수문자키 등)를 제공할 때 다음 1가지 방법 이상으로 제공하지 않은 경우
      1. 1)단축키를 비활성화하는 기능을 제공
      2. 2) 단축키를 재설정할 수 있는 기능을 제공
      3. 3) 단축키를 초점을 받은 상태에서만 실행되도록 구현

    14. (응답시간 조절)

    시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다.

    1. 14-1 시간제한이 있는 콘텐츠를 제어할 수 있는 수단을 제공하지 않은 경우
    2. 14-2 시간제한을 해제 또는 연장하는 방법에 충분한 시간을 제공하지 않은 경우

    15. (정지 기능 제공)

    자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다.

    1. 15-1 시간에 따라 자동으로 변경되는 콘텐츠에 정지, 이전, 다음 기능이 없는 경우
    2. 15-2 시간에 따라 자동으로 변경되는 콘텐츠를 마우스와 키보드 또는 터치로 제어할 수 없는 경우

    16. (깜빡임과 번쩍임 사용 제한)

    초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠 를 제공하지 않아야 한다.

    1. 16-1 사전 경고 없이 초당 3~50회 깜빡이는 콘텐츠가 존재할 경우

    17. (반복 영역 건너뛰기)

    콘텐츠의 반복되는 영역은 건너뛸 수 있어야 한다.

    1. 17-1 건너뛰기 링크를 제공하지 않은 경우
    2. 17-2 건너뛰기 링크의 제공 방법이 적절하지 않은 경우

    18. (제목 제공)

    페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다.

    1. 18-1 페이지에 제목을 제공하지 않은 경우
    2. 18-2 페이지에 제목을 적절하게 제공하지 않은 경우
    3. 18-3 프레임 요소에 제목을 제공하지 않은 경우
    4. 18-4 프레임 요소에 제목을 적절하게 제공하지 않은 경우
    5. 18-5 콘텐츠 블록에 제목을 제공하지 않은 경우
    6. 18-6 콘텐츠 블록에 제목을 적절하게 제공하지 않은 경우

    19. (적절한 링크 텍스트)

    링크 텍스트는 용도나 목적을 이해할 수 있도록 제공해야 한다.

    1. 19-1 목적이나 용도를 알기 어려운 링크 텍스트를 제공한 경우

    20. (고정된 참조 위치 정보)

    전자출판문서 형식의 웹 페이지는 각 페이지로 이동할 수 있는 기능이 있어야 하고, 서식이나 플랫폼에 상관없이 참조 위치 정보를 일관되게 제공ㆍ유지해야 한다.

    1. 20-1 웹페이지로 제공되는 전자출판문서 콘텐츠에 대한 페이지 구분 정보를 제공하지 않은 경우
    2. 20-2 웹페이지로 제공되는 전자출판문서 콘텐츠에 대한 페이지 이동 기능을 제공하지 않은 경우
    3. 20-3 콘텐츠 변경 (글꼴 변경, 간격 조절 등)시 일관된 페이지 구분 정보를 제공·유지하지 않은 경우

    21. (단일 포인터 입력 지원)

    다중 포인터 또는 경로기반 동작을 통한 입력은 단일 포인터 입력으로도 조작할 수 있어야 한다.

    1. 21-1 다중 포인터 또는 경로 기반 동작을 통한 입력을 수행할 때 동등한 단일 포인터 입력을 함께 사용할 수 없는 경우

    22. (포인터 입력 취소)

    단일 포인터 입력으로 실행되는 기능은 취소할 수 있어야 한다.

    1. 22-1 단일 포인터 입력으로 실행되는 기능을 취소할 수 있는 기능을 다음 1가지 방법 이상으로 제공하 지 않은 경우
      1. 1) 누르는 동작(Down-Event)에 실행되지 않도록 금지 기능 제공
      2. 2) 완료 전에 기능을 중단하거나 완료 후 기능을 실행 취소하는 기능을 제공
      3. 3) 떼는 동작(Up-Event)에 누르는 동작의 결과를 되돌리는 기능 제공

    23. (레이블과 네임)

    텍스트 또는 텍스트 이미지가 포함된 레이블이 있는 사용자 인터페이스 구성요소는 네임에 시각적으로 표시되는 해당 텍스트를 포함해야 한다.

    1. 23-1 시각적으로 보이는 텍스트 레이블(label)에 대한 접근 가능한 네임(name)을 포함하지 않은 경우

    24. (동작기반 작동)

    동작기반으로 작동하는 기능은 사용자 인터페이스 구성요소로 조작할 수 있고, 동작기반 기능을 비활성화할 수 있어야 한다.

    1. 24-1 동작 기반 작동에 대한 비활성화 기능을 제공하지 않은 경우
    2. 24-2 버튼 등 사용자 인터페이스 구성요소로 대체 기능을 제공하지 않은 경우

  3. 이해의 용이성(Understandable) : 콘텐츠는 이해할 수 있어야 한다.

    25. (기본 언어 표시)

    주로 사용하는 언어를 명시해야 한다.

    1. 25-1 문서 타입에 맞는 기본 언어를 명시하지 않은 경우
    2. 25-2 문서 타입에 맞는 기본 언어를 적절하지 않게 명시한 경우

    26. (사용자 요구에 따른 실행)

    사용자가 의도하지 않은 기능(새 창, 초점에 의한 맥락 변화 등)은 실행되지 않아야 한다.

    1. 26-1 사용자가 실행하지 않은 상황에서 예측하지 않은 새 창이 열리는 경우
    2. 26-2 웹 페이지 시작 시 새 창 또는 화면을 가리는 레이어 팝업을 제공하는 경우
    3. 26-3 사용자가 의도하지 않은 초점 변화나 기능이 발생되어 맥락상 불편을 주는 경우
    4. 26-4 입력 서식의 값을 변경하는 것만으로 다른 페이지로 이동하거나 현재 페이지의 의미가 완전히 바뀌는 경우

    27. (찾기 쉬운 도움 정보)

    도움 정보가 제공되는 경우, 각 페이지에서 동일한 상대적인 순서로 접근할 수 있어야 한다.

    1. 27-1 각 웹페이지에서 다음과 같은 사용자 도움 정보를 서로 다른 위치에 제공한 경우
      1. - 담당자 상세 연락처: 전화번호, 이메일, 운영시간 등
      2. - 담당자 연락 방법: 메신저, 채팅창, 게시판, SNS 등
      3. - 도움말 옵션: FAQ, 사용법 등 - 자동화된 연결 방법: 챗봇 등

    28. (오류 정정)

    입력 오류를 정정할 수 있는 방법을 제공해야 한다.

    1. 28-1 입력 오류 발생 시 오류 원인 및 내용을 적절한 설명 텍스트로 알려주지 않거나, 적절하지 않은 경우
    2. 28-2 입력 오류 발생 시 정정할 수 있는 수단을 제공하지 않은 경우
    3. 28-3 오류 정보 확인 시 해당 입력 서식으로 초점이 이동하지 않는 경우
    4. 28-4 입력 오류 발생 시 입력 내용이 모두 사라지는 경우

    29. (레이블 제공)

    사용자 입력에는 대응하는 레이블을 제공해야 한다.

    1. 29-1 <input>, <textarea>, <select> 요소에 1:1 대응하는 <label> 요소 또는 title 속성을 제공하지 않거나 설명이 적절하지 않은 경우
    2. 29-2 <input>의 id 속성값과 <label>의 for 속성값이 다르거나, 페이지 안에 같은 id가 있는 경우

    30. (접근 가능한 인증)

    인증 과정은 인지 기능 테스트에만 의존해서는 안 된다.

    1. 30-1 사용자 인증 과정에서 인지 기능 테스트(로그인을 위한 비밀번호 입력, 터치스크린 화면의 패턴 인식, 임의의 문자열 기억, 특정 객체를 포함한 이미지 찾기, 퍼즐 풀기 등)에 의존할 때, 인지 기능 테스트에 의존하지 않는 인증방식을 함께 제공하지 않은 경우

    31. (반복 입력 정보)

    반복되는 입력 정보는 자동 입력 또는 선택 입력할 수 있어야 한다.

    1. 31-1 반복되는 입력 정보에 대한 자동 입력 또는 선택 입력 기능을 제공하지 않은 경우

  4. 견고성(Robust) : 웹 콘텐츠는 미래의 기술로도 접근할 수 있도록 견고하게 만들어야 한다.

    32. (마크업 오류 방지)

    마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다.

    1. 32-1 태그의 열고 닫음 오류
    2. 32-2 태그의 열고 닫는 순서가 중첩되는 오류
    3. 32-3 중복 선언된 속성 오류
    4. 32-4 한 페이지 안에 동일 ID 값이 존재하는 경우

    33. (웹 애플리케이션 접근성 준수)

    콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 한다.

    1. 33-1 웹 애플리케이션이 자체적인 접근성이 없으며, 사용자가 선택할 수 있는 대체 콘텐츠가 존재하지 않거나 적절하지 않게 제공한 경우

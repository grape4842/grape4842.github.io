1. 적절한 대체 텍스트 제공

이미지의 대체 텍스트(alt) 속성이 생략되어 있습니다. 대체 텍스트 속성 자체를 생략하게 되면 이미지 요소의 파일 경로 명을 스크린 리더가 읽게 됩니다.

이미지가 전달하고자 하는 정보를 올바르게 대체 텍스트 속성을 통해 제공해야합니다.

이미지에 대체 텍스트가 제공되지 않아 스크린리더 사용자는 이미지에 담긴 내용을 알 수 없어 이미지를 통해 전달받는 정보에 차등이 생기게 됩니다.

이미지에 담긴 내용을 스크린리더 사용자도 알 수 있도록 대체 텍스트를 제공해야 합니다.

이미지의 대체 텍스트 속성값이 실제 이미지의 내용과 다르게 제공되어 스크린리더 사용자는 이미지에 담긴 내용을 알 수 없어 이미지를 통해 전달받는 정보에 차등이 생기게 됩니다.

스크린 리더 사용자가 이미지의 내용을 올바르게 전달받을 수 있도록 대체 텍스트를 이미지와 동일한 내용으로 수정해야 합니다.

이미지에 대체 텍스트가 제공되지 않아 스크린리더 사용자는 이미지에 담긴 내용을 알 수 없어 이미지를 통해 전달받는 정보에 차등이 생기게 됩니다.

그러나 이미지에 담긴 내용을 대체 텍스트로 제공하기에는 이미지의 내용이 복잡하답면 이를 숨김 텍스트로 제공하여 스크린 리더 사용자가 일반 텍스트와 같이 내용을 쉽게 탐색할 수 있도록 해야 합니다.

이미지에 제공된 대체 텍스트가 실제 이미지의 내용과 비교했을 때 대체 텍스트가 생략되어 있어 스크린 리더 사용자는 이미지를 통해 전달받는 정보에 차등이 생기게 됩니다.

대체 텍스트를 이미지에 담긴 정보와 동일한 내용을 충분히 담을 수 있도록 수정해야 합니다.

이미지에 제공된 대체 텍스트에 오타가 있어 스크린리더 사용자는 이미지에 담긴 내용과는 다른 내용을 전달받게 되어 이미지를 통해 전달받는 정보에 차등이 생기게 됩니다.

스크린 리더 사용자가 이미지의 내용을 제대로 전달받을 수 있도록 대체 텍스트를 올바르게 수정해야 합니다.

이미지의 대체 텍스트 속성값이 실제 이미지의 내용과 다르게 제공되어 스크린리더 사용자는 이미지와는 다른 잘못된 내용을 전달받게 됩니다. 이미지의 내용을 올바르게 전달받을 수 있도록 수정해야 합니다.

그러나 이미지에 담긴 내용을 대체 텍스트로 제공하기에는 이미지의 내용이 복잡하고 많습니다. 따라서 숨김 텍스트로 제공하여 스크린 리더 사용자가 일반 텍스트와 같이 내용을 쉽게 탐색할 수 있도록 해야 합니다.

숨김텍스트 제공 방법

이미지가 통이미지 형태로 제공되었으나 대체 텍스트가 제공되지 않아 스크린 리더 사용자는 이미지의 내용을 확인할 수 없습니다.

이미지가 (콘텐츠 내용을 숨김 텍스트로 제공하기 어려운 경우 (이미지를 문자가 포함된 PDF로 저장하여 / 원본 hwp, word를) 첨부파일로 대체 제공할 수 있습니다.)

이미지의 대체 텍스트를 숨김 텍스트로 제공하였으나 실제 이미지와 달리 많은 내용이 그저 텍스트로 영역구별 없이 제공되어 내용을 논리적, 구조적으로 알기 어렵습니다.

실제로 제공된 이미지와 유사하게 사용자가 내용을 파악하기 쉽도록 태그를 사용하여 내용을 구조에 맞게 나누어 제공해야 합니다.

대체 텍스트와 동일한 내용이 이미 텍스트로 충분히 제공됨에도 이미지에 동일한 대체 텍스트가 제공되어 중복된 내용을 제공받게 됩니다. 또한 이미지에 대체 텍스트가 제공되었기 때문에 불필요한 탐색도 추가로 발생하게 됩니다.

따라서 스크린 리더가 이미지를 탐지하지 않고 자동적으로 넘어갈 수 있도록 대체 텍스트의 속성값("alt")을 공백으로 제공해야 합니다.

<img>태그명("이미지")를 대체 텍스트로 제공하였기에 "이미지"가 중복으로 출력됩니다.

이미지의 대체 텍스트에 포함되어 있는 중복된 정보("이미지")를 삭제해야합니다.

이미지맵을 사용한 이미지의 내용이 대체 텍스트로 충분히 설명되지 않아 이미지맵으로 제공된 내용을 스크린 리더 사용자는 알 수 없읍니다.

이미지맵을 통해 표현한 콘텐츠의 내용을 스크린 리더 사용자도 정확하게 알 수 있도록 대체 텍스트를 충분히 제공해야 합니다.

표를 이미지로 제공하여 표의 내용을 이해할 수 없습니다.

동일한 내용의 표를 Excel파일로 제공하는 것과 같은 대체 수단을 제공하거나 <table>태그를 숨긴 형태로 제공하여 스크린 리더 사용자도 표의 내용을 이해할 수 있도록 제공해야 합니다.

QR코드는 이미지로 제공되기 때문에 스크린리더 사용자, 키보드 사용자는 이미지를 이해하거나 접근할 수 없어 QR코드에 제공된 기능이나 경로를 이용할 수 없습니다.

모든 사용자가 QR코드의 제공된 정보를 이용할 수 있도록 대체 수단을 제공해야 합니다.

필수입력 항목을 안내하는 기호에 대한 설명이 존재하지 않습니다. 또한 스크린 리더 설정에 따라 기호가 구분되어 음성 출력되지 않을 수 있습니다.

따라서 특수 기호로 필수항목에 대해 안내한 것에 대해 별도로 대체 텍스트를 제공하여 스크린 리더 사용자도 해당 항목이 필수항목임을 알 수 있도록 제공해야 합니다.

장식을 위한 이미지에 대체 텍스트 속성값이 제공되어 스크린리더 사용자에게 의미 없는 이미지의 대체 텍스트 정보로 불필요한 탐색을 추가로 요하게 됩니다.

정보를 담고 있지 않고 장식으로 제공되는 이미지는 스크린 리더에서 탐지하지 않고 자동적으로 넘어갈 수 있도록 대체 텍스트의 속성값("alt")을 공백으로 제공해야 합니다.

의미를 전달하기 위한 콘텐츠를 CSS 가상 선택자로 표현하였으나 이에 대한 대체 콘텐츠를 제공하지 않아 스크린 리더 사용자는 해당 콘텐츠가 의미하는 바를 알 수 없습니다.

가상 선택자를 텍스트로 변경하거나 대체 텍스트로 별도의 설명을 제공하여 스크린 리더 사용자도 해당 콘텐츠를 이해할 수 있도록 제공해야 합니다.

이미지의 전·후에 제공된 텍스트를 통해 이미지에서 전달하고자 하는 정보를 충분히 파악할 수 있습니다.

이 때문에 사용자의 탐색 편의성을 위해 대체 텍스트 속성을 빈 값으로 수정해야 합니다.

배경으로 제공한 이미지에 정보가 있음에도 이에 대한 대체 텍스트 정보를 제공되지 않아 스크린리더 사용자는 이미지의 내용을 알 수 없습니다.

스크린리더 사용자가 배경으로 제공된 이미지의 내용을 알 수 있도록 숨김 텍스토로 정보를 제공해야 합니다.

배경으로 제공한 이미지에 정보가 있어 이에 대한 대체 텍스트 정보를 제공하였으나 이미지와는 다른 잘못된 정보가 제공되어 스크린리더 사용자는 이미지의 내용을 알 수 없습니다.

스크린리더 사용자가 배경으로 제공된 이미지의 내용을 올바르게 알 수 있도록 대체 텍스트를 수정해야 합니다.

사용자가 이미지를 업로드 할 수 있도록 기능이 제공되고 있으나 업로드한 이미지에 대해 대체 텍스트를 입력하여 제공할 수단이 제공되지 않았습니다.

이미지와 함께 대체 텍스트를 제공할 수 있도록 기능을 구현하여 제공해야 합니다.

별도의 첨부파일로 제공된 PDF파일이 이미지 형태로 제공되어 파일의 내용을 스크린리더로 확인할 수 없습니다.

PDF파일을 제공할 경우 PDF의 파일의 내용을 스크린 리더 사용자도 확인할 수 있또록 텍스트를 확인할 수 있는 형태로 제공해야 합니다.

숨김 텍스트가 제공되었지만 스크린리더가 인지할 수 없는 형태로 제공되어 결과적으로 대체 텍스트가 제공되지 못하였습니다.

스크린리더가 인지할 수 있는 형태로 숨김 텍스트 제공방법을 변경하여 대체 텍스트가 제공될 수 있도록 수정해야 합니다.

스크린 리더 사용자를 위해 숨김 텍스트를 제공하였으나 이에 적용된 CSS가 부적절한 형태로 기술되어 결과적으로 스크린 리더 사용자는는 정보를 전달받을 수 없는 상태입니다.

visibility:hidden, display:none, font-size:0, opacity:0 중 하나 이상이 적용된 경우, 스크린 리더가 정보에 접근할 수 없게 되므로 대체 텍스트를 위해 지정한 선택자에 해당 속성이 기술되어 있는 경우 삭제해야 합니다. text-indent 역시 사용 시 시각적으로 표시되는 요소의 위치와 상관 없는 위치 또는 뷰포트를 넘어가는 영역에 스크린 리더 가상 초점을 생성시키므로 사용하지 않아야 합니다.

올바른 숨김텍스트 제공 코드

대체 텍스트가 title속성을 통한 잘못된 방식으로 선언되었습니다. title은 마우스 오버시에 표시되는 메시지 속성으로 대체 텍스트가 아닙니다. 따라서 title속성으로 정보를 제공한 것으로는 대체 텍스트가 제공되지 않기 때문에 스크린 리더 사용자는 이미지에 담긴 정보를 인지할 수 없습니다.

따라서 이미지의 정보를 인지할 수 있도록 대체 텍스트 속성(alt)으로 대체 텍스트를 제공해야 합니다.

2. 자막 제공

영상의 음성, 내부 텍스트 정보를 담은 그림 등에 대해 자막,원고가 제공되지 않아 영상의 내용을 알 수 없습니다.

자막, 원고, 수화를 제공하여 모든 사용자가 영상의 내용을 알 수 있도록 대체 콘텐츠를 제공해야 합니다.

영상의 음성, 내부 텍스트 정보를 담은 그림 등에 대해 자막,원고가 실제 영상과는 다른 내용으로 잘못 제공되어 영상의 내용을 알 수 없습니다.<

자막, 원고, 수화를 수정하여 모든 사용자가 영상의 내용을 알 수 있도록 대체 콘텐츠를 올바르게 제공해야 합니다.

사용자가 영상을 업로드할 수 있도록 기능을 제공하고 있으나 자막, 원고, 수화를 함께 압럳,할 수단이 제공되지 않았습니다.

사용자가 영상과 함께 자막, 원고, 수화 또한 업로드하여 제공할 수 있도록 기능을 구현하여 제공해야 합니다.

3. 색에 무관한 콘텐츠 인식

콘텐츠의 내용의 차이를 색상만으로 표현했기 때문에 색상을 구별할 수 없는 사용자의 경우, 콘텐츠의 차이를 알 수 없습니다.

색상뿐만이 아닌 다른 방법(선, 굵기, 그림자 등)으로 콘텐츠의 차이를 구별할 수 있도록 수정해야 합니다.

~의 선택여부가 색상 이외의 수단으로 구분할 수 없는 형태로 제공되어 색각이상, 스크린 리더 사용자는 현재 어느것이 선택되어 있는지 알 수 없습니다.

색상 외의 시각적으로 구분할 수 있는 형태(밑줄, 크기 차이, 1px 이상의 테두리 차이, 표식자 등)로 선택여부를 알 수 있도록 제공해야 합니다.

(색상만으로 선택여부를 알 수 있도록 제고하고자 할 경우 7:1이상의 명도대비를 준수하여 제공해야합니다.)

그래프의 범례를 색상만으로 구분하고 있어 색각이상, 저시력 사용자는 그래프의 내용을 알 수 없습니다.

그래프의 각 범례를 형태, 크기, 패턴(예 - 가로/세로/대각선 채움) 또는 명도(명도 대비 7:1 이상)를 다르게 주는 것으로 각각의 범례가 다른 것임을 알 수 있도록 제공해야 합니다.

그래프의 범례를 색상만으로 구분하고 있어 색각이상, 저시력 사용자는 그래프의 내용을 이해하기 어려우며 스크린 리더 사용자는 그래프를 이해할 수 없습니다..

그래프와 동일한 내용의 표를 추가로 제공하는 것으로 모든 사용자가 그래프의 내용을 이해할 수 있도록 해야합니다.

8. 키보드 사용 보장

상호작용이 불가능한 요소(<>)에 Java Script로 클릭 이벤트를 연결하여 키드로는 기능을 실행할 수 없도록 잘못된 방법으로 구현되었습니다.

컨트롤 요소를 키보드로 접근 및 조작할 수 있도록 상호작용이 가능한 태그(<a>, <button> 등)로 변경해야 합니다.

커스텀 UI를 키보드로 접근 및 조작할 수 없으며 스크린 리더 사용자는 해당 요소가 무엇을 표현하는지 알 수 없습니다.

커스텀 UI를 키보드로 상호작용이 가능한 요소로 마크업을 변경해야 하며 스크린 리더 사용자가 해당 UI가 어떤 요소인지 예측하고 조작할 있도록 WAI-ARIA 등의 보조 기술을 활용하여 요소를 원활하게 사용할 수 있도록 제공해야 합니다.

링크<a> 태그에 필수 속성인 href 속성이 선언되어 있지 않아 컨트롤 요소로 인식되지 못해 키보드로 접근할 수 없습니다.

링크가 컨트롤 요소로 인식되어 키보드로 접근할 수 있도록 href 속성을 선언해야 합니다.

마우스로 오버(:hover)한 상태인 경우에만 스타일을 기술하였기 때문에 키보드 사용자는 동일한 콘텐츠에 접근했을 때 전달받는 기능이나 정보에 차이가 생기게 됩니다.

스타일 시트에 :focus 선택자를 추가하여 키보드로 접근한 경우에도 마우스와 동일한 효과가 발생하여 이를 통해 키보드 사용자도 동일한 기능과 정보를 제공받을 수 있도록 구현해야 합니다.

스크린 리더로 접근시 해당 요소가 기능이 있는 요소임을 알 수 없음에도 실제로는 기능이 제공되고 있어 조작할 수 있는 요소임을 예측할 수 없습니다.

요소의 역할을 알 수 있도록 수정하여 스크린 리더 사용자가 조작할 수 있는 요소임을 알고 조작할 수 있도록 수정해야 합니다.

기능이 제공되는 요소에 스크린리더가 접근할 수 없습니다.

코드를 통해 오류 원인을 파악하고 스크린 리더로 해당 요소에 초점이 접근할 수 있도록 수정해야 합니다.

tabindex="-1"을 제거하여 키보드가 접근할 수 있도록 수정해야 합니다.

9. 초점 이동

컨트롤을 활성화 하여 레이어가 팝업되었을 떄 초점이 레이어 팝업 자체 또는 레이어 팝업 내 첫 번째 요소로 이동하지 않아 스크린 리더 사용자는 레이어가 팝업된 사실을 알 수 없습니다

레이어가 팝업 된 사실을 알 수 있도록 레이어가 팝업되었을 때 초점이 레이어 내로 초점이 이동하도록 구현해야 합니다.

레이어 팝업을 닫았을 때, 초점이 레이어가 팝업되도록 한 컨트롤로 이동되도록 구현해야 합니댜.

컨트롤에 (outline:none / opacity:0) CSS가 적용되어 있어 키보드로 사이트 탐색시 초점을 시각적으로 확인할 수 없어 키보드 사용자는 현재 자신의 위치를 알 수 없어 사이트를 탐색할 수 없습니다.

키보드 사용자가 시각적으로 초점의 위치를 확인할 수 있도록 해당 스타일을 제거해야 합니다.

페이지가 로딩되자마자 제공되는 레이어 팝업이 <body>하위 태그 중 가장 첫번째로 마크업되어 있지 않아 다른 영역에 키보드 초점이 먼저 접근하기 때문에 키보드 사용자는 레이어를 닫지 못해 본문 가려진 본문 콘텐츠를 알 수 없습니다.

페이지 로딩 시 본문 콘텐츠를 덮는 레이어 팝업을 사용하는 경우 레이어를 <body>태그의 자식 중 첫번째(건너뛰기 링크 이전)로 마크업하여 키보드 사용자가 레이어를 닫을 수 있도록 구현해야 합니다.

시각을 활용하여 웹 페이지를 탐색하는 사용자가 콘텐츠에 접근하는 순서와 초점이 접근하는 순서가 다릅니다. 이렇게 되면 키보드 사용자가 다음 초점의 위치를 예측하기 어려워 이용하는데 있어 혼란을 겪을 수 있습니다.

논리적인 구조상의 초점 순서 조정을 제외하고 시각적인 콘텐츠의 흐름과 동일한 순서대로 키보드 초점이 콘텐츠에 접근하도록 수정해야 합니다.

페이지 내부 이동 컨트롤 실행 시, 목표한 위치로 화면은 스크롤되나 초점이 컨트롤에 멈춰 있어 스크린 리더 ,키보드 사용자는 해당 기능을 사용할 수 없어 페이지 내 영역이동 시 불편을 겪게 됩니다;.

컨트롤을 실행했을 때 설정한 지점으로 키보드 초점이 함께 이동할 수 있도록 수정해야 합니다.

컨트롤을 실행하여 새로운 콘텐츠가 추가되었을 때 초점이 페이지 처음으로 튀는 오류가 발생하여 키보드 사용자가 페이지를 처음부터 탐색해야 하기 때문에 사이트 이용에 혼란을 겪게 됩니다.

컨트롤을 실행하여 콘텐츠가 새로 생성되면 새로 생성된 콘텐츠 중 첫 번째 콘텐츠로 초점을 이동시켜 스크린 리더 사용자가 콘텐츠가 새롭게 추가되었음을, 키보드 사용자가 추가된 요소에서 탐색을 시작할 수 있도록 수정해야 합니다

페이지 처음으로 돌아가는 컨트롤을 실행했을 떄 키보드 초점이 페이지 가장 처음이 아닌 다른 영역으로 이동하여 키보드 및 스크린 리더 사용자는 페이지를 처음부터 탐색할 수 있는 수단이 제공되지 않았습니다.

컨트롤의 목적지를 수정하여 문서 가장 처음으로 이동할 수 있게 하여 키보드 및 스크린 리더 사용자도 페이지를 처음부터 탐색할 수 있는 수단을 제공해야 합니다.

컨트롤 실행 시 키보드 초점이 예기치 못하게 튀는 오류가 발생하여 키보드 및 스크린 리더 사용자는 해당 컨트롤을 다시 찾아가야 하기 때문에 사이트 이용에 혼란을 겪게 됩니다.

컨트롤을 실행했을 때 초점이 컨트롤에 남아있도록 수정하여 예기치 못하게 초점이 튀는 일이 없도록 수정해야 합니다.

자동으로 갱신되는 콘텐츠에 초점이 접근한 경우 초점이 예기치 못하게 튀는 오류가 발생하여 키보드 및 스크린 리더 사용자는 해당 컨트롤을 다시 찾아가야 하기 떄문에 사이트 이용에 혼란을 겪게 됩니다.

컨트롤에 접근했을 때 초점이 컨트롤에 남아있도록 수정하여 예기치 못하게 초점이 튀는 일이 없도록 수정해야 합니다.

재생/정지 버튼 실행 시 버튼에 display:none 속성이 적용되면서 키보드 초점이 소실되어 키보드 및 스크린 리더 사용자는 예기치 못한 초점 이동으로 혼란을 겪게 됩니다.

누른 버튼이 사라지는 동시에 나타나는 버튼으로 초점이 이동하도록 focus 이벤트를 할당하여 키보드 초점이 예기치 못한 곳으로 이동하지 않도록 구현해야 합니다.

표시된 요소에 키보드 초점이 진입하게 되면 키보드 초점이 해당 영역에서 빠져나가지 못하는 키보드 함정 현상이 발생하여 tab, shift + tab을 눌러 이전/다음 요소로 키보드 탐색을 실행할 수 없기 때문에 키보드 및 스크린 리더 사용자는 더 이상 사이트를 탐색할 수 없습니다.

스크립트를 확인하여 해당 요소에 부적절한 keystroke 이벤트가 연결되어 있지 않은지 확인하고 해당 현상이 발생하지 않도록 수정해야 합니다.

tabindex 속성값이 0이 아닌 다른 숫자가 사용되었습니다. 이러한 경우 키보드 초점은 낮은 숫자 속성값을 가진 순서대로(1부터) 접근하게 되어 초점의 이동 순서가 비논리적으로 마크업 순서를 무시하게 됩니다.

불필요한 tabindex 속성을 제거하여 초점이 논리적으로 이동할 수 있도록 해야 합니다.

tabindex속성을 사용하지 않을 경우 초점이 비논리적으로 이동할 경우 마크업 순서를 변경해야 합니다.

링크를 담고 있는 영역에 선언된 overflow:hidden 및 인접 영역과의 중첩으로 인해 키보드 초점을 시각적으로 명확하게 확인할 수 없습니다.

링크의 outline-offset 스타일을 -2px ~ -4px 수준으로 조정하여 사용자가 시각적으로 키보드 초점을 명확하게 인식할 수 있도록 수정해야 합니다.

링크 요소가 크기를 갖지 않아 키보드로 사이트 탐색시 초점을 시각적으로 확인할 수 없어 키보드 사용자는 현재 자신의 위치를 알 수 없어 탐색을 진행할 수 없습니다.

표의 제목셀에 scope속성이 제공되지 않아 스크린리더 사용자가 표를 탐색할 경우 데이터 셀이 표의 제목 셀의 값을 참조하지 않아 표를 이해하기 어려움. scope속성을 제공하여 표를 탐색하기 쉽도록 수정해야 함.
20-4. 다단, 병합 등 복잡한 표를 제공 시
headers, id 속성을 통해 제목 셀과 내용 셀을 연결하지 않은 경우
표 안에 표를 제공하여 표의 내용을 이해하기 어려움. 표가 중첩되지 않도록 마크업을 수정해야 함.

입력서식에 제공된 label/title이 적절하지 않아 스크린리더 사용자는 입력서식의 목적을 이해할 수 없으므로 입력서식의 목적이 무엇인지 정확히 이해할 수 있도록 label/title을 수정해야 함.
입력서식에 제공된 label/title에 필수여부가 포함되지 않아 스크린리더 사용자는 입력서식의 필수 여부를 제공받지 못하므로 스크린리더 사용자도 해당 입력서식이 필수항목임을 알 수 있도록 label/title을 수정해야 함.

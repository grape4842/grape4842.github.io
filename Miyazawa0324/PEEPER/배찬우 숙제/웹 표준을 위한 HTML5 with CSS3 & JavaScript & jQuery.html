  Sction 01. HTML 기본 문법 익히기
기본개념: ‘HTML’은 ‘Hyper Text Markup Language’의 약어로 하이퍼링크 기능을 포함하여 텍스트나 이미지 등을 웹브라우저에 표시할 수 있게 하는 마크업 언어를 말합니다.
여러분은 요즘 궁금한 것이 있으면 제일 먼저 누구에게 물어보시나요? 혹시 아직도 무거운 백과사전을 찾아보시나요? 21C에 살고 있는 지금 우리에게 없어서는 안 될 공간이 바로 인터넷입니다. 
찾고 싶은 자료를 검색하고, 쇼핑하고, 홍보하고, 수많은 시스템을 제어하는 일 등 오프라인에서 이루어지는 모든 일이 바로 여러분이 보고 있는 모니터 앞 인터넷이라는 세상에서 이루어지고 있으니까요.
세계의 수많은 공공기관, 회사, 단체, 개인들이 자신이 원하는 바를 홍보하고 알리기 위해 이젠 인터넷이라는 세상에 자신의 공간을 만들어 놓았습니다. 
웹 브라우저에 해당 주소만 입력하면 바로 여러분과 연결이 되지요. 이것이 바로 홈페이지입니다.
이 홈페이지의 구조를 이루는 기본이 바로 HTML이란 언어입니다. 
HTML로 작성된 웹 문서는 인터넷만 연결되어 있으면 간단한 클릭만으로 쉽게 페이지를 이동하면서 원하는 수많은 정보를 얻을 수 있게 합니다. 
이렇게 웹상에서 페이지와 페이지를 이동할 수 있도록 하는 것을 하이퍼링크라고 하는데, HTML(Hyper Text Markup Language)은 이러한 하이퍼링크 기능을 포함하여 텍스트나 이미지 등을 웹 브라우저에 표시할 수 있도록 합니다.
HTML은 다양한 분야에서 활용되고 있습니다. 
그 중 가장 HTML이 비중 있게 사용되고 있는 분야는 홈페이지일 것입니다. 
이번 장에서는 HTML의 기본 구조와 문법 그리고 HTML의 기본적인 태그들이 홈페이지를 제작하는데 있어 어떻게 적용되는지 살펴보겠습니다.
1.1. HTML 기본 문법구조 파악하기
HTML 태그의 기본 형식은 다음과 같이 태그(Tag)와 내용(content, 데이터)으로 구성되어 있습니다.
[형식]
〈태그명 속성＝"값" 속성＝"값" 속성＝"값". . . .〉내용〈/태그명〉
[형식 끝]
4
태그란 ‘〈 〉’ 두 괄호 안에 설정된 명령어로서, HTML 문서의 모양이나 행동양식을 정해주는 HTML의 구성요소라 할 수 있습니다. 
태그는 ‘〈 〉’로 둘러싸여 다른 문장들과 구별되며, 서로 쌍을 이루는 형식으로 되어 있습니다.
여기서 〈태그명〉이 시작을 알리는 역할을 하고 〈/태그명〉이 끝을 알리는 역할을 해주어 태그의 적용범위가 끝났음을 알려줍니다. 
〈태그명〉과 〈/태그명〉사이의 브라우저에 실제로 출력될 내용을 입력하면 됩니다. 
HTML은 대소문자의 구별이 없으며, 태그를 사용할 때 주의사항은 태그를 교차하여 사용할 수는 없다는 점입니다. 
또한 모든 태그들이 종료태그를 갖고 있지는 않습니다. 
태그는 종류에 따라 별도의 속성을 가지고 있습니다. 
태그의 속성은 시작 태그에서 지정합니다. 여러 개의 속성을 사용할 수 있으며 속성의 구분은 공백으로 합니다. 
속성의 값을 입력할 때는 인용 부호로 큰따옴표(" ") 혹은 작은따옴표(' ')를 이용합니다. 
하지만 HTML5에서는 인용부호를 생략할 수 있습니다.
1.2. HTML의 기본구조
홈페이지를 만드는 것은 태그를 이용해서 HTML 문서를 만드는 것을 의미합니다. 
텍스트나 이미지, 동영상 등의 다양한 요소들이 HTML 태그에 의해서 웹 브라우저에 전달될 수 있습니다. 
다음은 HTML 문서를 작성하기 위해서 기본적으로 사용하는 태그들을 살펴보도록 합시다.
HTML의 기본구조는 DOCTYPE과 〈HTML〉, 〈HEAD〉, 〈TITLE〉, 〈BODY〉 4가지 기본 태그로 이루어져 있으며 DOCTYPE과 문자 인코딩을 지정해야 합니다.
[HTML의 기본구조]
01: 〈!DOCTYPE html〉
02: 〈html〉
03: 〈head〉
04: 〈meta charset＝"UTF-8"〉
05: 〈title〉 html 문서의 제목이 들어갑니다.〈/title〉

1.3. DOCTYPE이란?
DOCTYPE은 어떤 유형으로 만들어진 것인지에 대해 작성한 HTML 문서의 유형을 정의하는 요소입니다. 
작성된 HTML 문서가 HTML4.01 포맷인지, XHTML1.0 포맷인지 HTML5 포맷인지를 DOCTYPE 선언으로 알려줍니다. 
DOCTYPE에서 지정한 유형에 의해서 HTML에서 사용한 마크업이 유효한 태그나 속성인지를 판단합니다. 
유효하지 않은 HTML 코드는 개발자가 원치 않는 결과를 웹 브라우저로 볼 수 있기 때문입니다.
다음은 HTML5 이전 버전에서 사용한 DOCTYPE 중에서 가장 많이 사용한 형태입니다.
☑ 〈!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"〉
HTML4에서는 문서의 종류, HTML 버전 등 매우 긴 DOCTYPE 선언을 정의했습니다. 
이러한 DOCTYPE 형식을 기억하는 사람은 없을 것입니다.
HTML5에서 DOCTYPE은 아주 간단한 형태로 정의합니다. 
어떤 브라우저에서도 표준 모드로 랜더링할 것으로 간주하기 때문입니다.
☑ 〈! DOCTYPE html〉
DOCTYPE은 반드시 대문자로 작성해야 하며, 생략하면 브라우저가 최신 랜더링 모드를 사용하지 않아 HTML5의 일부 기능이 작동하지 않습니다.
1-4. 문자 인코딩이란?
인코딩은 문자셋을 컴퓨터가 이해할 수 있는 바이트와의 매핑 규칙을 말합니다. 
예를 들면 ASCII Code에서 ABC 등은 문자셋이고 A는 코드 65, B는 코드 66 등 바이트 순서와 매핑한 것이 인코딩입니다. 
따라서 문자셋을 어떻게 매핑하느냐에 따라 하나의 문자셋이 다양한 인코딩을 가질 수 있습니다.
컴퓨터는 영미권에서 만들어졌기 때문에 영어를 표현하는 경우는 아무런 문제가 없지만 다른 문자를 사용하는 나라에서 자국의 언어로 표현하려면 문제가 되는 경우가 있습니다. 
그렇기 때문에 한글을 표현하는 인코딩을 적용하여 문서를 만들어야 합니다. 
한글 인코딩에는 MS949(윈도 OS에서 기본으로 사용), UTF-8 등이 있습니다.
6
웹 브라우저는 내부적으로 모두 유니코드로 처리합니다. 
그렇기 때문에 HTML5에서는 문자 인코딩에 UTF-8을 권장하고 있습니다. 
UTF-8은 전 세계적으로 모두 통용될 수 있는 표준화된 텍스트 데이터를 표현하기 위해서 만들어진 인코딩 방식입니다.
UTF-8 방식을 이용하면 다음과 같이 표기합니다.
☑ 〈meta charset＝"UTF-8"〉
또한 MS949 방식을 이용하면 다음과 같이 표기합니다.
☑ 〈meta charset＝"EUC-KR"〉
기존의 사용 방식인 〈meta http-equiv＝"Content-Type" content＝"text/html； charset＝UTF-8"〉에서 변경하는 것이 좋습니다.

1-5. HTML 문서를 구성하는 기본 태그 4가지
이제 본격적으로 HTML의 구조를 이루는 4가지의 기본 태그들에 대해서 살펴보겠습니다.
4가지로 대표되는 HTML 태그는 머리 부분(head)과 몸통 부분(body)으로 나뉩니다. 
머리 부분은 HEAD 태그로 둘러싸인 부분이고 몸통 부분은 BODY 태그로 둘러싸인 부분입니다. 
4개의 기본 태그들의 역할을 개별적으로 살펴보도록 합시다.
(1) HTML 문서의 시작과 종료(〈html〉~〈/html〉)
〈html〉~〈/html〉은 지금 작성하고 있는 문서가 HTML 언어로 만들어졌다는 것을 알려주며, HTML 문서의 시작과 끝을 명시해 줍니다. 
웹에 있는 거의 모든 HTML 문서는 반드시 〈html〉로 시작해서 〈/html〉로 끝납니다.
(2) HTML 문서의 머리 부분(〈head〉~〈/head〉)
〈head〉 태그는 해당 HTML 문서의 일반적인 정보를 표시하는 일종의 머리말 역할을 합니다. 이 태
7
그는 〈title〉 태그를 포함하는데, 〈title〉 태그 내부에 입력한 내용은 웹 브라우저의 화면 상단에서 문서의 제목을 보여줍니다. 
타이틀(title)을 제외하고는 HEAD 부분에 기술한 내용은 웹 브라우저를 통해 사용자에게 보이지 않습니다. 
또한 자바스크립트나 스타일 시트로 문서의 형태를 정의할 때도 〈head〉 태그 안에 기술할 수 있습니다.
(3) HTML 문서의 타이틀(〈title〉~〈/title〉)
〈title〉 태그에 기술한 내용은 웹 브라우저의 화면 상단에 나타나며, HTML 문서의 제목(타이틀)을 지정하는 태그입니다. 
이 태그에는 64문자 이내로 타이틀을 기입하는 것이 좋습니다. 
HTML 문서의 타이틀만 보고도 문서의 내용을 알아낼 수 있도록 간단하고 명료한 문구로 정하는 것이 좋습니다.
(4) HTML 문서의 몸통 부분(〈body〉~〈/body〉)
〈body〉와 〈/body〉 안에 기술한 내용은 실질적인 내용이 들어가는 곳으로, 웹 브라우저 화면에 바로 나타나게 됩니다. 
〈head〉 부분에 들어가는 태그를 제외한 모든 태그는 〈body〉 안에 기술하여 사용합니다. 
웹 브라우저를 통해 보여지는 HTML 문서들은 BODY 부분에 기술한 각 태그의 내용들입니다.

2-2. 문단 나누는 〈p〉 태그
〈p〉 태그는 HTML에서 문단을 나누어 _단락_을 생성할 때 사용합니다.

단락이라는 의미는 줄을 바꾼다는 의미와 함께 새로운 서식이 시작된다는 의미이기도 합니다.
